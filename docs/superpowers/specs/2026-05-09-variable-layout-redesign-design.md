# 变量布局编辑器重设计

## 目标

将任务编辑中的「变量与布局」编辑器从「先布局后绑定变量」改为「变量先行」模式，去掉左侧组件面板，改为双区拖拽模式。

## 布局结构

- **左侧 (200px)**：「待布局变量」面板，列出 `values` 中未出现在 `layout` 里的变量
  - 每个变量卡片显示：变量名、默认值、类型预览图标
- **右侧**：「布局画布」，已放置的变量以可视化卡片展示
  - 每行 24 列网格
  - 空行/空位显示可拖入提示

## 交互流程

### 拖入布局
1. 从左侧拖拽变量卡片到右侧布局区
2. 剩余空间 >= 12 时高亮可拖入
3. 剩余空间 < 12 时该行灰显 + 提示「空间不足」
4. 放下后弹出「选择控件类型」弹窗
5. 弹窗按变量默认值类型过滤兼容控件
6. 选择控件后变量加入布局，默认 span = 12

### 布局内拖拽排序
- 已布局卡片可拖拽换位（同行或跨行）
- 拖拽时显示插入指示线

### 右键菜单
- 右键已布局卡片 → 编辑属性 / 更换控件 / 调整宽度 / 移除
- 移除后变量回到左侧待布局列表，值保留

### 无按钮添加
- 唯一的添加方式是拖拽

## 类型过滤规则

仅显示兼容的控件 — 按 JS 类型严格匹配：

| 默认值类型 | 兼容控件 |
|-----------|---------|
| string | el-input, el-textarea, el-select, el-date-picker, el-color-picker, el-radio, el-checkbox-group |
| number | el-input-number, el-slider, el-select, el-radio |
| boolean | el-switch, el-checkbox |
| 空/undefined | 全部 |

## 控件选择弹窗

- 网格展示兼容控件卡片
- 每张卡片：类型图标 + 中文名 + 带入默认值的 MiniPreview
- 不可用控件自动隐藏并底部提示

## 文件变更范围

1. **重写** `src/pages/task-editor/LayoutBuilder.tsx` — 双区拖拽布局
2. **新建** 控件选择弹窗组件（可内联于 LayoutBuilder 或独立文件）
3. **新建** 类型过滤工具函数
4. **保留** `MiniPreview` 组件逻辑，复用

## 不涉及

- `TaskVarsSection.tsx` — 旧版变量编辑，本次不动
- `VariablePanel.tsx` — 变量参考面板，不动
- 后端 API
